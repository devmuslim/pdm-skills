# مراجعة Figma — عام لأي مشروع

## متى تُفعَّل
عند مراجعة تصميم Figma مقابل متطلبات أو BRD.

---

## منهجية المراجعة

### 1. إذا تم تحليل BRD مسبقاً:
- تحقق من كل Use Case — هل الشاشات تغطيه؟
- تحقق من الحقول — Mandatory / Optional / Read Only
- تحقق من الرسائل — هل نصوص الإشعارات صحيحة؟
- تحقق من حالات الخطأ — هل مصممة؟

### 2. إذا لم يوجد BRD:
راجع التصميم بناءً على:
- **اتساق الواجهة** — ألوان، خطوط، مسافات
- **تجربة المستخدم** — وضوح التدفق
- **الحالات الناقصة** — Empty states، Loading، Error states
- **Accessibility** — تباين الألوان، أحجام النص
