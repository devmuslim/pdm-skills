# System Thinking — رؤية النظام كله

## المبدأ الجوهري
```
كل مشكلة تراها هي عَرَض — وليست المشكلة الحقيقية.
Delivery Lead يرى الجبل بأكمله، لا قمته فقط.
```

---

## أداة 1 — Iceberg Model

```
ما يُرى (10%):
┌─────────────────────────────┐
│  السلوك / الحدث             │  ← ما يقوله الناس
│  "Dev دائماً يتأخر"         │
└─────────────────────────────┘

ما لا يُرى (90%):
┌─────────────────────────────┐
│  الأنماط                    │  ← متى يحدث؟ كيف يتكرر؟
│  "التأخير يبدأ دائماً بعد  │
│   Refinement"               │
├─────────────────────────────┤
│  الهياكل                    │  ← ما الذي يُنتج الأنماط؟
│  "Stories تدخل Sprint       │
│   بدون DoR واضح"           │
├─────────────────────────────┤
│  المعتقدات / Mental Models  │  ← الجذر الحقيقي
│  "Dev يعتقد أن Requirements │
│   ستتغير على أي حال"        │
└─────────────────────────────┘

التدخل الصحيح: عند الهياكل والمعتقدات — لا عند السلوك
```

## أداة 2 — Causal Loop Diagram (5 Whys للـ Leads)

```
السؤال الذي يغير كل شيء:
"لماذا يحدث هذا؟" × 5

مثال — صراع بين Dev وQA:

Why 1: لماذا يتشاجر Dev وQA؟
→ QA يرفض Stories ويعيدها

Why 2: لماذا يرفضها؟
→ Bugs كثيرة + AC غير واضحة

Why 3: لماذا AC غير واضحة؟
→ Refinement لا يشمل QA

Why 4: لماذا QA مستبعد من Refinement؟
→ "QA مهمته Verify لا Define"

Why 5: من قرر ذلك؟
→ ثقافة قديمة — لا أحد قررها رسمياً

الجذر: ثقافة Siloed — QA ليس شريكاً في التعريف
الحل: Three Amigos في كل Story
```

## أداة 3 — Leverage Points (نقاط التأثير)

```
أين تتدخل لأكبر تأثير بأقل جهد؟

منخفض التأثير ❌:
- إضافة قواعد جديدة
- تحذيرات ومتابعة يومية
- "سنجرب لأسبوعين"

متوسط التأثير ⚠️:
- تغيير العملية (Process)
- تغيير الهيكل التنظيمي

عالي التأثير ✅:
- تغيير المعلومات المتاحة للناس
- تغيير الأهداف (Goals)
- تغيير المعتقدات (Mental Models)
- تغيير البيانات والـ Feedback Loops
```

## أداة 4 — Unintended Consequences

```
قبل أي قرار، اسأل:
"ماذا لو نجح هذا الحل بشكل مثالي؟
ما الذي سيسوء بسببه؟"

أمثلة شائعة:
┌────────────────────┬───────────────────────────────┐
│ القرار             │ العاقبة غير المتوقعة           │
├────────────────────┼───────────────────────────────┤
│ زيادة Velocity     │ تراجع الجودة                  │
│ ضغط على الـ Timeline│ Burnout خفي                  │
│ إضافة عمليات أكثر │ الناس تتحايل عليها            │
│ مكافأة السرعة      │ Technical Debt يتراكم         │
└────────────────────┴───────────────────────────────┘
```

## كيف تطبق System Thinking في موقف صعب

```
الخطوة 1: صِف الحدث بجملة واحدة
الخطوة 2: ابحث عن الأنماط — متى يتكرر؟ مع من؟
الخطوة 3: ابحث عن الهياكل — ما العملية التي تنتج هذا؟
الخطوة 4: اكشف المعتقدات — ماذا يؤمن الناس المتورطون؟
الخطوة 5: حدد Leverage Point — أين تتدخل؟
الخطوة 6: اسأل عن العواقب غير المتوقعة
```
