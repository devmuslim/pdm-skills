# Skill: sprint-review-template

## Purpose
Structure a sprint review that showcases delivered value, collects meaningful feedback, and feeds the next sprint — not just a demo session.

## When to Use
- End of every sprint
- Preparing for stakeholder demo

## Framework: Sprint Review Structure (60 Minutes)

**5 min — Sprint Summary**
- Sprint goal — was it achieved? YES / PARTIALLY / NO
- Planned vs delivered (story points and key stories)
- Carry-overs and reason

**20 min — Demo**
- Show working software only (no slides for features)
- Demo Must Have stories first
- Include stakeholders as active participants: "Can you try clicking X?"

**15 min — Stakeholder Feedback**
- What did stakeholders love?
- What questions do they have?
- What would they like to see next?

**10 min — Backlog Review**
- What's coming next sprint (top 5 items)
- Any reprioritization based on feedback?

**10 min — Metrics**
- Velocity trend (last 3 sprints)
- Release burndown (how many sprints left)
- Any risks to the release

## Output Format
1. Sprint review agenda (ready to share with attendees)
2. Demo script per story
3. Feedback capture template
4. Suggested next command: `/groom-backlog` or `/retrospective`
