## Agile Maturity Model

```
المستوى 1 — Chaotic (الحالة الأولى)
لا عملية واضحة، reactive، كل شيء urgent

المستوى 2 — Defined ← هدف الشهر 3
عملية واضحة، Ceremonies منتظمة، Backlog منظم

المستوى 3 — Measured ← هدف الشهر 6
قياس مستمر، KPIs واضحة، تحسين مبني على البيانات

المستوى 4 — Optimizing ← هدف السنة الأولى
فريق ذاتي التنظيم، يحسن عمليته بنفسه
```

## علامات نجاح التحول

### 🟢 علامات إيجابية
- الفريق يطرح أسئلة في Backlog Refinement
- يكتشفون gaps في المتطلبات قبل التطوير
- يقترحون تحسينات في Retrospective بشكل تلقائي
- يساعدون بعضهم بدون طلب
- يفكرون في User Value لا في Tasks فقط

### 🔴 علامات تحتاج تدخل
- لا أحد يتكلم في Daily Standup إلا إذا سُئل
- نفس المشاكل تتكرر في كل Retrospective
- Stories تنتهي Sprint بدون إغلاق متكرر
- لا أحد يقرأ BRD قبل البدء
