# تشخيص مستوى الفريق — Team Assessment

## الهدف
قياس المستوى الحالي لكل عضو في 5 محاور قبل بناء خطة التطوير.

---

## محاور التقييم (كل محور من 1 إلى 5)

### 1. 🧠 فهم Agile & Scrum
| المستوى | الوصف |
|---------|-------|
| 1 | لا يعرف Agile أو Scrum |
| 2 | سمع عنه لكن لم يطبقه |
| 3 | يفهم المبادئ الأساسية |
| 4 | يطبقه ويعلّم غيره |
| 5 | خبير، يكيّف الـ framework |

### 2. ⚡ سرعة التنفيذ
| المستوى | الوصف |
|---------|-------|
| 1 | يحتاج توجيه مستمر لإنهاء المهام |
| 2 | يكمل المهام لكن بتأخير متكرر |
| 3 | يكمل في الوقت المحدد في الغالب |
| 4 | سريع ويساعد الفريق |
| 5 | يكمل قبل الموعد ويأخذ مهام إضافية |

### 3. 🔍 فهم المتطلبات
| المستوى | الوصف |
|---------|-------|
| 1 | ينفّذ بدون فهم السياق |
| 2 | يفهم المطلوب لكن يفوته تفاصيل |
| 3 | يفهم ويسأل الأسئلة الصحيحة |
| 4 | يكتشف gaps في المتطلبات |
| 5 | يساهم في كتابة Acceptance Criteria |

### 4. 💻 جودة الكود
| المستوى | الوصف |
|---------|-------|
| 1 | الكود يعمل لكن غير منظم |
| 2 | يتبع بعض المعايير |
| 3 | كود نظيف ومقروء |
| 4 | يطبق Design Patterns |
| 5 | يرفع مستوى الفريق بالـ Code Review |

### 5. 🤝 التواصل والتعاون
| المستوى | الوصف |
|---------|-------|
| 1 | يعمل بشكل منعزل |
| 2 | يتواصل عند الحاجة فقط |
| 3 | يشارك في Ceremonies بفاعلية |
| 4 | يساعد زملاءه ويشارك المعرفة |
| 5 | يقود النقاشات ويحل النزاعات |

## تفسير المجموع

| المجموع | المستوى | الخطة المناسبة |
|---------|---------|--------------|
| 5–10 | 🔴 مبتدئ | تأسيس شامل — 3 أشهر |
| 11–15 | 🟡 متوسط | تطوير مستهدف — شهرين |
| 16–20 | 🟢 متقدم | صقل وقيادة — شهر |
| 21–25 | 🌟 خبير | مرشح للـ Mentoring |
