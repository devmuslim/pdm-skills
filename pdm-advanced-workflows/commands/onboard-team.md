# Command: /onboard-team

## الوصف
خطة إدماج متكاملة لعضو جديد في فريق التسليم — من اليوم الأول إلى الأسبوع الرابع.

## الاستخدام
```
/onboard-team [اسم العضو] [دوره] [اسم الفريق]
```

## أمثلة
```
/onboard-team سارة — مطوّرة Backend — فريق Platform
/onboard-team أحمد — QA Engineer — فريق Mobile
/onboard-team نورا — Scrum Master — فريق Data
```

---

## Workflow — 4 أسابيع

### الأسبوع 1 — التوجيه والفهم

**اليوم 1:**
- [ ] إعداد الوصول: Jira, Confluence, Slack, GitHub, بيئة التطوير
- [ ] لقاء ترحيب مع الفريق (15 دقيقة)
- [ ] تسليم دليل الفريق (Working Agreements)
- [ ] تعيين Buddy من داخل الفريق

**اليوم 2-3:**
- [ ] مراجعة الـ Roadmap الحالي
- [ ] قراءة آخر 3 تقارير حالة
- [ ] مراجعة آخر Sprint Retrospective
- [ ] فهم بنية النظام / المنتج

**اليوم 4-5:**
- [ ] حضور جميع اجتماعات الفريق كمراقب
- [ ] مراجعة الـ Backlog الحالي
- [ ] لقاء 1-on-1 مع مدير التسليم

**هدف الأسبوع 1:** فهم السياق، التعرف على الفريق، إعداد البيئة.

---

### الأسبوع 2 — المشاركة التدريجية

- [ ] المشاركة في Sprint Planning (مع توجيه)
- [ ] تولّي مهمة صغيرة واضحة النطاق
- [ ] حضور Three Amigos session
- [ ] مراجعة Definition of Done
- [ ] لقاء مع أصحاب المصلحة الرئيسيين

**هدف الأسبوع 2:** أول مساهمة فعلية في التسليم.

---

### الأسبوع 3 — الاستقلالية المُوجَّهة

- [ ] تولّي مهمة كاملة باستقلالية
- [ ] المشاركة في Bug Triage
- [ ] تحديث وثيقة أو عملية موجودة
- [ ] تقديم Sprint Review update
- [ ] تقييم منتصف الطريق (30 دقيقة مع PDM)

**هدف الأسبوع 3:** الاستقلالية في المهام الأساسية.

---

### الأسبوع 4 — الاندماج الكامل

- [ ] تولّي مسؤولية كاملة لجزء من الـ Backlog
- [ ] قيادة اجتماع واحد على الأقل
- [ ] تقديم تقرير حالة أسبوعي بشكل مستقل
- [ ] تقييم نهاية الشهر الأول (1 ساعة)
- [ ] وضع أهداف الأشهر الثلاثة القادمة

**هدف الأسبوع 4:** عضو فاعل ومستقل في الفريق.
