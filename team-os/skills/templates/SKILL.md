## 📋 User Story Template

```
العنوان: [فعل + هدف — مثال: إضافة تعديل العقد]

كـ [نوع المستخدم]
أريد [الإجراء / الميزة]
لكي [القيمة / الهدف التجاري]

📏 Story Points: [1 / 2 / 3 / 5 / 8 / 13]
🔗 BRD Reference: [BRD-XXX]
🎨 Figma: [رابط]

✅ Acceptance Criteria:
Given [السياق]
When  [الإجراء]
Then  [النتيجة]

(أضف AC لكل سيناريو: Happy Path + Error + Edge Cases)

📎 التبعيات:
- يعتمد على: [Story / API / فريق آخر]
- يؤثر على: [X]

📝 ملاحظات تقنية (اختياري):
[أي قيود أو اعتبارات تقنية]
```

## 🐛 Bug Report Template

```
العنوان: [مكان المشكلة] — [وصف قصير]
مثال: صفحة التعديل — لا يُحفظ التعديل عند النقر على حفظ

🔴 الأولوية: P1 / P2 / P3 / P4
🌍 البيئة: Dev / Test / Staging / Production
📅 تاريخ الاكتشاف: [X]
👤 اكتشفه: [الاسم]

📸 خطوات إعادة الإنتاج:
1. اذهب إلى [X]
2. افعل [Y]
3. لاحظ [Z]

✅ النتيجة المتوقعة:
[ماذا يجب أن يحدث]

❌ النتيجة الفعلية:
[ما يحدث فعلاً]

📎 الأدلة:
- Screenshot: [رابط]
- Video: [رابط]
- Logs: [رابط]

🔗 BRD Reference: [BRD-XXX]
🔗 مرتبط بـ Story: [TICKET-XXX]

📊 التأثير:
- كم مستخدم متأثر؟
- هل هناك Workaround؟
```

## 🤝 Three Amigos Template

```
الموضوع: Story [TICKET-XXX]
الحضور: BA [اسم] + Dev [اسم] + QA [اسم]
التاريخ: [X]

أسئلة BA:
- ما القيمة التجارية من هذه الـ Story؟
- ما الحالات التي لم نفكر فيها؟

أسئلة Developer:
- ما التعقيد التقني؟
- هل هناك بدائل أبسط؟
- ما التبعيات التقنية؟

أسئلة QA:
- كيف سنختبر هذا؟
- ما Edge Cases المحتملة؟
- هل AC كافية للاختبار؟

القرارات:
- [قرار 1]
- [قرار 2]

AC المضافة / المعدّلة:
- [AC جديدة]
```
