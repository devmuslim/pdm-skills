# Team Health — صحة الفريق

## نموذج Spotify Squad Health Check

### 8 محاور التقييم

| المحور | 🔴 ضعيف | 🟡 متوسط | 🟢 ممتاز |
|--------|---------|---------|---------|
| **Easy to Release** | تسليم صعب ومؤلم | أحياناً صعب | سلس وآلي |
| **Suitable Process** | العملية تعيق | مقبولة | تساعدنا فعلاً |
| **Tech Quality** | ديون تقنية عالية | مقبول | نفخر بكودنا |
| **Value** | لا نعرف قيمة عملنا | نحدس | نرى التأثير |
| **Speed** | بطيئون جداً | مقبول | نسلّم بسرعة |
| **Mission** | لا هدف واضح | نعرف المهمة | ملتزمون بالمهمة |
| **Fun** | لا نستمتع | أحياناً | نحب عملنا |
| **Learning** | لا نتعلم | نتعلم قليلاً | نتعلم باستمرار |

---

## طريقة التطبيق

```
1. كل عضو يقيّم سراً كل محور: 🔴 / 🟡 / 🟢
2. SM يجمع النتائج ويعرضها للفريق
3. ناقشوا المحاور الحمراء
4. Action Item لكل محور أحمر
5. كرر كل شهر وتابع التحسن
```

---

## مؤشرات صحة الفريق من Jira

```
Happiness Index:     تصويت الفريق (1-5) كل Retro
Velocity Stability:  هل Velocity ثابت أم متذبذب؟
Sick Days Rate:      ارتفاع مفاجئ = مؤشر إرهاق
After-hours Commits: عمل خارج الدوام = مؤشر ضغط
Bug Rate Trend:      ارتفاع مستمر = مؤشر إهمال الجودة
Turnover Rate:       خروج أعضاء = مؤشر بيئة سيئة
```

---

## علامات التحذير المبكرة

```
🚨 تحتاج تدخل فوري:
- عضو يعمل باستمرار وحده بدون تواصل
- خلافات شخصية متكررة في الـ Retro
- لا أحد يتكلم في Daily Standup
- Velocity انهار فجأة > 30%
- أكثر من عضو يفكر في المغادرة

⚠️ مراقبة مستمرة:
- نفس Action Items تتكرر كل Retro
- Retro تحول لجلسة شكاوى
- PO لا يحضر Review
- Stories تعود من QA باستمرار
```
