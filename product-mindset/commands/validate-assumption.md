# Command: /validate-assumption
## Usage: `/validate-assumption [صف الـ Feature + ما وصلك من BA/PM عنها]`
## Workflow: product-discovery + experimentation + context-bridge → كشف الافتراضات + أصغر تجربة

## Description
يكشف الافتراضات الخفية في أي Feature — خاصة تلك التي تبدو "واضحة" لأن BA قالها.
يقترح أسرع طريقة للتحقق قبل إهدار وقت التطوير.

## Steps
1. اجلب skills/product-discovery/SKILL.md
2. اجلب skills/experimentation/SKILL.md
3. اجلب skills/context-bridge/SKILL.md
4. فرّق بين: ما قاله BA (مباشر) + ما افترضه (ضمني)
5. استخرج كل الافتراضات من كلا المصدرين
6. صنّفها بالمصفوفة: مهمة × غير مثبتة
7. اقترح أصغر تجربة لكل افتراض خطير

## Output Format
```

📋 ما قاله BA/PM (مباشر):
- [X]

🔍 ما لم يُقل لكن افتُرض (ضمني):
- [Y] — مصدره: [منطق / تخمين / تجربة سابقة]

⚠️ الافتراضات الخطيرة (مهمة + غير مثبتة):
```
