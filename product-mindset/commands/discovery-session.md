# Command: /discovery-session
## Usage: `/discovery-session [المشكلة أو المجال + ما تعرفه من BA/PM حتى الآن]`
## Workflow: product-discovery + context-bridge + customer-empathy → أجندة جلسة كاملة

## Description
يصمم جلسة Discovery تجمع BA وDev وQA معاً — لا لتوزيع المهام
بل لبناء فهم مشترك للمشكلة قبل أي حديث عن الحلول.
يعالج الفجوة بين ما يعرفه BA وما يحتاجه الفريق التقني.

## Steps
1. اجلب skills/product-discovery/SKILL.md
2. اجلب skills/context-bridge/SKILL.md
3. اجلب skills/customer-empathy/SKILL.md
4. حدد الهدف: ماذا نريد أن نعرف بعد الجلسة؟
5. صمّم أجندة تجمع BA/PM بالفريق التقني
6. اكتب أسئلة تُخرج السياق من BA وتبنيه للـ Dev
7. حدد المخرجات الملزمة وخطوات ما بعد الجلسة

## Output Format
```
🔍 جلسة Discovery — [الموضوع]
الهدف: [ما نريد معرفته بعد الجلسة]
الحضور: BA + Dev + QA (+ PM اختياري)
المدة: [X دقيقة]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📋 الأجندة:
[5 دق]  الافتتاح — "لماذا نحن هنا"
         BA يشرح: المستخدم + المشكلة + لماذا الآن
[15 دق] أسئلة الفريق للـ BA/PM
         [قائمة الأسئلة أدناه]
[15 دق] Assumption Mapping مشترك
         "ما الذي نفترضه بدون دليل؟"
[10 دق] تحديد الأولويات
         أهم 3 افتراضات نحتاج نثبتها
[5 دق]  الإغلاق — Context Card الأولية

❓ أسئلة الفريق للـ BA:
للمستخدم:
1. "من بالضبط سيستخدم هذا؟ صفه لنا"
2. "كيف يحل هذه المشكلة الآن بدوننا؟"
3. "ما أسوأ شيء يحدث له إذا لم نبنِ هذا؟"

للأولويات:
4. "لماذا هذا Sprint تحديداً؟"
5. "ما الذي نتنازل عنه إذا بنيناه الآن؟"

للنجاح:
6. "كيف سنعرف بعد شهر أننا نجحنا؟"

🎯 المخرجات الملزمة (لا ننهي الجلسة بدونها):
[ ] Context Card مكتملة
[ ] North Star Metric متفق عليه
[ ] أهم افتراض سنختبره قبل البناء
[ ] من المسؤول عن إيصال Feedback المستخدم للـ Dev؟

📋 خطوات ما بعد الجلسة:
[ ] BA يكمل Context Card في Jira — اليوم
[ ] Dev يراجعها ويؤكد أنها كافية — غداً
[ ] اختبار الافتراض الأهم — [موعد]
```
